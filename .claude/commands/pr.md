# Pull Request 作成コマンド

## 事前チェック

1. **コードフォーマット**: `task format` を実行してコードを整形
2. **コード品質チェック**: `task check` を実行して品質を確認
   - リンター、型チェック、テストが全て成功することを確認
   - エラーがある場合は修正してから次へ進む

## PR 作成

- gh コマンドを使用して Pull Request を作成
- マージ先は基本的にベースとなったブランチにする
- PR のタイトルと説明を適切に記載する
  - タイトルには prefix と github emoji をつける
  - 例 `feat: ✨ 実装内容`
- 使用する emoji は `.claude/commands/gitemoji.json` を参照する
- テストプランを記載する
- 実装内容の概要を簡潔に説明する
- 必要に応じて、PR にラベルを付与する。
    - ラベルは基本的に標準的な、既存のものを使用する
    - 適切なものが無い場合は、ユーザーに確認して新規作成する
